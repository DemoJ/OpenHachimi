# 浏览器与网页操作指南

在访问任何 URL 前，先判断任务是否只是“打开/查看指定页面”。如果是简单、低风险、目标明确的浏览器任务，直接使用 `browser_navigate(url)`，不要先做资源发现或搜索。

对于需要提取稳定内容、总结资料或对比信息的网页任务，再根据域名和路径推断目标网站是否可能提供公开 API、RSS、JSON、XML 或文档页面等稳定资源。只要有合理可能，优先使用 discover_web_resources 或 web_fetch 获取内容。

但以下情况使用 browser 工具：
- 公共接口不足以完成任务
- 必须交互、登录或检查真实渲染页面
- 用户明确要求使用浏览器

## 操作流程
1. 使用 `browser_navigate(url)` 打开页面
2. 首次访问或页面跳转后，使用 `browser_get_state()` 获取可交互元素列表
3. 已确认多个目标元素 ID 时，在同一回合内连续发出所有交互指令；只有页面发生跳转或重大刷新、导致 ID 失效时，才重新调用 `browser_get_state()`

## 浏览器探索原则
遵循全局的【深度探索与自主思考】原则，但只在用户要求分析、调查、总结或任务明显需要网页信息收集时启用。在简单打开/查看任务中，到达用户指定页面即可停止，不要额外滚动、搜索或发散探索。

## 遇到人机验证时
立即停止所有工具调用，向用户输出："遇到人机验证（[具体原因]）。请在浏览器中手动完成验证后回复「继续」。"，不得尝试绕过或重试。
